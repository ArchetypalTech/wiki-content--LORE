
`Exit` is not just a pointer to another room. It is an interactive component with command dispatch, validation, movement, and optional post-move action triggers.

## Purpose

The `Exit` component turns an entity into a traversable connection between areas. It maps verbs like `go`, `enter`, or `use` into exit behavior.

## Model

The `Exit` Dojo model is composed by:

```
#[dojo::model]
pub struct Exit {
    #[key]
    pub inst: felt252,
    pub is_exit: bool,

    /// Properties ///
    /// If the exit is enterable
    pub is_enterable: bool,

    /// The leads to entity
    pub leads_to: felt252,

    /// The direction type
    pub direction_type: Direction,

    /// Array of action maps for the exit
    pub action_map: Array<ActionMapExit>,
}
```

### Variables Characteristics

- `inst`: Is the unique ID of the entity/object to whom the component is attached to.
- `is_enterable`: Indicates whether traversal is currently allowed.

- `leads_to`: Indicates the target object where it leads to. The target should posses an area component.

- `direction_type`: Indicates the direction the exit object leads to.

- `action_map`: Links the verb-to-function mapping for command parsing when interacting with the object:

```
pub struct ActionMapExit {
    /// The action verb
    pub action: ByteArray,

    /// The inst of the component that the action is attached to
    pub inst: felt252,

    ///  The types of actions
    pub action_fn: ExitActions,
}

pub enum ExitActions {
    #[default]
    UseExit,
}
```

- `action`: The verb that can be used to interact with the object.
  
- `inst`: The `inst` of the exit component.
  
- `action_fn`:  The type of action to be executed when interacting with the objec through the verb.
  
- `UseExit`: Type of function that when called through the verb will execute the logic of using it as an exit. 
	- Examples: `use the door`, `enter the telepor`, `go north`
